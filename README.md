# 📧 Email → Telegram Bot v4.0

Професійний інструмент для пересилання IMAP-пошти в Telegram з підтримкою розширеної аналітики, спам-фільтрації та керування кількома акаунтами.

---

## ✨ Основні можливості

- **📬 Розумне пересилання**: Миттєве отримання листів з будь-якого IMAP-провайдера (UKR.NET, Gmail, Outlook тощо).
- **📂 Мультипапковість**: Автоматичний моніторинг усіх папок (не тільки INBOX), окрім спаму та кошика.
- **🛡️ Advanced Spam Filter**: Багаторівнева перевірка (ключові слова, патерни, аналіз відправника, блокування за доменом).
- **✉️ Інтерактивні відповіді**: Можливість відповідати на листи прямо з Telegram-чату.
- **📊 Аналітичний дашборд**: Веб-панель зі статистикою, піковими годинами та трендами.
- **📱 Telegram Mini App**: Зручний інтерфейс для керування налаштуваннями всередині Telegram.
- **🔕 Режим спокою**: Автоматичне приглушення сповіщень у нічний час (черга повідомлень приходить зранку).
- **🔐 Безпека**: Шифрування паролів додаткових акаунтів (AES-256) та захист API-експорту.

---

## 🚀 Швидкий старт

1. **Клонуйте репозиторій**:
   ```bash
   git clone <repository_url>
   cd email-telegram-bot
   ```

2. **Налаштуйте середовище**:
   ```bash
   cp .env.example .env
   # Відредагуйте .env, вказавши свої дані
   ```

3. **Встановіть залежності та запустіть**:
   ```bash
   npm install
   npm start
   ```

## ⚙️ Змінні середовища (.env)

| Змінна | Приклад | Опис |
|--------|---------|------|
| `EMAIL_USER` | `user@ukr.net` | Основна пошта для моніторингу |
| `EMAIL_PASSWORD` | `your_app_password` | Пароль дотатка (або основний) |
| `TELEGRAM_BOT_TOKEN` | `123456:ABC...` | Токен від @BotFather |
| `TELEGRAM_CHAT_ID` | `645718667` | ID чату, куди прийдуть листи |
| `IMAP_HOST` | `imap.ukr.net` | Адреса IMAP-сервера |
| `ENCRYPTION_KEY` | `hex_string_32_bytes` | Ключ для шифрування дод. паролів |
| `QUIET_START` | `23` | Початок "тихого режиму" (година) |
| `QUIET_END` | `07` | Кінець "тихого режиму" |

## 🤖 Команди бота

| Команда | Функція |
|---------|---------|
| `/status` | Поточний стан з'єднань та аптайм |
| `/stats` | Коротко про кількість листів |
| `/analytics` | Детальні піки активності та тренди |
| `/digest` | Список листів за сьогодні |
| `/search <текст>` | Пошук у заголовках та відправниках |
| `/filter` | Керування білим списком/фільтрами |
| `/spam` | Перегляд заблокованого спаму |
| `/addmail` | Додати ще один акаунт для моніторингу |
| `/accounts` | Список усіх підключених пошт |

## 🌐 Веб-панель та API

Бот піднімає веб-сервер (порт 3000 за замовчуванням):
- `/` — Головна сторінка з дашбордом.
- `/miniapp` — Адаптована версія для Telegram Mini App.
- `/api/health` — Для моніторингу працездатності.
- `/api/export` — Експорт даних у CSV/JSON (вимагає `API_KEY`).

---

## ☁️ Деплой на Render.com

Проект повністю готовий до деплою на Render:
1. Підключіть репозиторій.
2. Виберіть тип сервісу **Web Service**.
3. Додайте всі змінні з `.env`.
4. Вкажіть `RENDER_EXTERNAL_URL` (ваш URL на render.com) для роботи системи **Keep-alive**, яка не дає веб-сервісу "заснути".

---

## 🛠️ Технічні деталі

- **UID Persistence**: Бот зберігає стан останнього прочитаного листа у `data/store.json`.
- **Render Fix**: При перезапуску на Render (де файлова система скидається), бот автоматично синхронізується з найостаннішого повідомлення, щоб уникнути дублювання старих листів.
- **Graceful Shutdown**: При зупинці бот встигає зберегти стан бази на диск.
